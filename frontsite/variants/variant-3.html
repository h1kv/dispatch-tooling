The building blocks

Twelve node types. One execution model.

Every node takes the previous output as input and emits a result for the next. Mix research, generation, human checkpoints, and branching on the same canvas.

Start

Entry point. Sets the task description plus the default provider and model.

Investigate

Researches with live tools — web_search and fetch_url — before continuing.

Plan

Turns input into a structured roadmap of phases and milestones.

Design

Produces a spec or architecture from the plan ahead of it.

Create

Generates the artefact — code, content, or drafts — from the spec.

Evaluate

Critiques the output for quality and flags what needs another pass.

Document

Produces docs, summaries, and changelogs from the run.

ReviewHUMAN

A human checkpoint. Pauses the chain for Approve or Reject.

ConditionBRANCH

Routes True or False on the previous output to split the path.

Tool Call

Hits an external API or webhook and feeds the response downstream.

Memory

A shared in-chain store any node can read from and write to.

File Write

Writes the output to disk at a path you choose.

Why DISPATCH.AI

Everything an agent run needs, on one surface.

Multi-provider, per node

Choose OpenAI, Anthropic, or Google — and any model — independently on each node. Route fast models where speed matters and frontier models where it counts.

openai · gpt-4o anthropic · claude-opus google · gemini

Live tools, not just text

Agents call real tools mid-run — search the web, fetch a URL, read and write files, list a directory, or run a shell command.

web_search fetch_url read_file write_file list_files shell_exec

Human-in-the-loop

Drop a Review node anywhere and the chain pauses for approval. Approve to continue, reject to stop — nothing ships without a yes.

Approve Reject

Branching and shared memory

Condition nodes route True or False down different paths. Memory nodes hold shared state every node in the graph can read and write.

if score > 0.8 → true · → false

Real-time execution you can watch

Every node shows its state — idle, running, done, paused, or error — and edges animate while data moves between them. No guessing where the run is.

idle running done error

Shape the goal before you run it.

  • Break a goal into ordered phases and milestones.
  • Surface risks early, before any node executes.
  • Carry the plan into a Plan node on the canvas.
