How it works

Build the graph. Press Run. Watch it execute.

No glue code, no orchestration framework. The canvas is the program — wiring defines order, and output flows down the edges.

01

Place & wire nodes

Drop nodes onto the infinite canvas and connect them top-to-bottom. The edge between two nodes is the contract: upstream output becomes downstream input.

02

Configure each step

Set a provider and model per node, grant live tools where you need them, and add Review checkpoints or Condition branches anywhere in the chain.

03

Run & observe

Press Run and the chain executes automatically. Nodes light up idle → running → done, edges animate while live, and a Review node pauses for your call.

The node library

Twelve node types. One execution model.

Every node does one job and passes a clean output to the next. Compose them into research pipelines, content factories, code generators, or review gates.

Investigate + live tools

Agents that actually reach out

Give a node real capabilities and it uses them mid-run — search the web, fetch a URL, read or write files, list a directory, or run a shell command.

web_search fetch_url read_file write_file list_files shell_exec
Condition Memory

Branch the path, share the state

Condition nodes route True / False on the previous output, so one graph handles many cases. Memory nodes read and write a shared store any node in the chain can reach.

True → Create False → Document

Tool Call

Calls an external API or webhook and folds the response back into the running chain.

Document

Write it down

Produces docs, summaries, or changelogs — and File Write commits them to disk.

Human-in-the-loop

The chain pauses
when it matters.

Drop a Review node anywhere and execution stops there, holding its upstream output for you. Approve to continue down the graph — or Reject to halt the run. Automation you can trust, because nothing irreversible ships without a person.

  • Pauses with full context — you see exactly what the previous node returned.
  • Approve resumes the run from the exact node it stopped on.
  • Reject stops the chain cleanly before any File Write or Tool Call fires.
