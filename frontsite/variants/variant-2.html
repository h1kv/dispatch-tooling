How it works

Three steps from blank canvas to a running chain.

01

Place nodes

Drag Start, Investigate, Plan, Create and more onto an infinite canvas. Each node is one step.

02

Wire top-to-bottom

Connect outputs to inputs. The edge defines the flow; the previous node's output becomes the next node's context.

03

Press Run

The chain executes automatically. Nodes light up idle → running → done; edges animate as data moves through.

Per-node models

Pick a provider and model for every single node.

Route a research step to one model and a drafting step to another. OpenAI, Anthropic, or Google — chosen per node, not per project. The chain handles the rest.

Live tools

Agents that reach beyond the prompt.

Investigate and Tool Call nodes can search the web, fetch URLs, read and write files, list directories, and run shell commands — so a node acts on real data, not a stale snapshot.

Human-in-the-loop

Pause for approval before the chain moves on.

Drop a Review node anywhere and the run halts there. Read the output, then Approve to continue or Reject to stop. Nothing irreversible happens without a person in the loop.

Branching + memory

Route on results. Carry state across the graph.

Condition nodes branch True / False on the previous output, so a low score loops back to Plan while a high one moves to Document. Memory nodes give every node a shared store to read from and write to.

The palette

Twelve node types. One canvas.

Every node does one job and hands its output to the next. Compose them however the work demands.

Start

Entry point. Sets the task and a default provider / model.

Investigate

Researches with live tools — web_search, fetch_url and more.

Plan

Turns input into a structured roadmap of phases and milestones.

Design

Produces a spec or architecture from the plan.

Create

Generates the artefact — code, content, or drafts.

Evaluate

Critiques output for quality before it moves on.

Document

Writes docs, summaries, and changelogs from the result.

Review

Human checkpoint. Pauses the chain for Approve / Reject.

Condition

Branches True / False on the previous node's output.

Tool Call

Hits an external API or webhook mid-chain.

Memory

Shared in-chain store every node can read and write.

File Write

Writes the output to disk as a final step.
