How it works

Three steps from idea to output.

No flowchart syntax to learn. Drop nodes, connect them, run the chain.

01

Place nodes

Drag Start, Investigate, Plan, Create, Evaluate and more onto the canvas. Each node owns its own provider, model and prompt.

+ Investigate · Plan · Create
02

Wire them

Connect nodes top to bottom to set the order. Add a Condition to branch True/False, or a Memory node to share state across the graph.

Start → Plan → Create → Review
03

Press Run

The chain executes automatically. Watch each node move from idle to running to done — edges animate as output flows downstream.

Node reference

Twelve nodes. One chain.

Every node does one job and hands its output to the next. Compose them into anything from a research brief to a full release pipeline.

Start entry

Entry point. Sets the task description and default provider / model for the chain.

Investigate tools

Researches with live tools — web_search, fetch_url.

Plan shape

Turns input into a structured roadmap of phases and milestones.

Design spec

Produces a specification or architecture from the plan.

Create build

Generates the artefact — code, content or drafts.

Evaluate critique

Critiques the previous output for quality before it moves on.

Document write

Produces docs, summaries and changelogs from the result.

Review human

Pauses the chain for a human Approve / Reject decision.

Condition branch

Routes True / False on the previous node's output.

Tool Call api

Hits an external API or webhook from inside the chain.

Memory state

Shared in-chain context store — read and write across nodes.

File Write disk

Writes the output to disk as a file.

Live tools available to agents: web_search · fetch_url · read_file · write_file · list_files · shell_exec

Real-time execution

Watch the chain think.

Every node reports its state as it runs — and edges animate while output flows from one node to the next. No black boxes, no guessing where a run stalled.

  • idle— waiting for upstream output
  • running— model or tool call in progress
  • done— output passed downstream
  • paused— Review node awaiting approval
  • error— halted, inspect and retry

Branching

Route on the result

A Condition node reads the previous output and sends the chain down a True or False path. Build retry loops, quality gates, and fallbacks without code.

Evaluate → Condition
├─ true → Document
└─ false → Create (retry)

Memory

Share state across the graph

Memory nodes hold context any downstream node can read or write — so a decision made early in the chain stays available all the way through.

write memory[brief] = Investigate.out
read memory[brief] → Document.ctx
